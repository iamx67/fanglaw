# Ассеты персонажа

В этой папке лежат ассеты редактора персонажа для сайта.

Обычная базовая модель остаётся в старых папках:

- `body/`
- `ears/`
- `tails/`
- `manes/`
- `cheeks/`
- `eyes/`
- `patterns/`

## Walking-вариация

Для новой walking-модели добавлена отдельная ветка:

- `walk/`

Смысл такой:

- старая модель остаётся как есть
- новые body-слои для ходьбы живут отдельно
- общие элементы не смешиваются с walking-специфичными

## Что класть в `walk/`

Новая структура:

- `walk/body/base/`
- `walk/body/shadows/`
- `walk/body/contours/`
- `walk/patterns/body/`

Сюда складываются только слои, которые нужно подгонять под walking-позу:

- корпус
- живот
- лапы
- body-паттерны, которые заходят на живот или лапы

## Какие имена файлов использовать

Внутри walking-папок лучше сохранять те же имена файлов, что и у обычной модели.

Примеры:

- `walk/body/base/bodybase.png`
- `walk/body/shadows/bodybaseshadow.png`
- `walk/body/contours/bodybaseco.png`
- `walk/patterns/body/eyeswhite.png`
- `walk/patterns/body/pattern12nose.png`
- `walk/patterns/body/pattern1.png`
- ...
- `walk/patterns/body/pattern23.png`

Это упростит будущее переключение между обычной и walking-моделью.

## Что обычно не нужно дублировать

Если элемент подходит к обеим позам, его можно оставить только в старой папке.

По текущей схеме это обычно:

- уши
- хвосты
- контуры хвостов
- тени хвостов
- грива
- щёки
- tail patterns

## Практический workflow

1. Скопировать body-ассеты из обычной модели в `walk/`.
2. Заменить только те PNG, которые не подходят к walking-позе.
3. Остальные общие элементы оставить в старых папках.

## Важно

Такая же структура создана и в:

- `client/assets/characters/walk/`

Если потом walking-вариант будет использоваться и в редакторе сайта, и в самой игре, папки уже будут совпадать по схеме.
