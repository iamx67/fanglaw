# Ассеты персонажа

В этой папке лежат обычные ассеты персонажа для клиента игры.

Текущая базовая модель хранится в старых папках:

- `body/`
- `ears/`
- `tails/`
- `manes/`
- `cheeks/`
- `eyes/`
- `patterns/`

## Новая walking-вариация

Под walking-модель добавлена отдельная ветка:

- `walk/`

Идея простая:

- обычная стойка остаётся в старых папках
- walking-модель получает отдельные body-ассеты в `walk/`
- общие элементы не дублируются без необходимости

## Что класть в `walk/`

В `walk/` имеет смысл складывать только те слои, которые завязаны на корпус, живот и лапы.

Текущая структура:

- `walk/body/base/`
- `walk/body/shadows/`
- `walk/body/contours/`
- `walk/patterns/body/`

## Какие имена файлов использовать

Внутри walking-папок лучше сохранять те же имена файлов, что и у обычной модели.

То есть:

- `walk/body/base/bodybase.png`
- `walk/body/shadows/bodybaseshadow.png`
- `walk/body/contours/bodybaseco.png`
- `walk/patterns/body/eyeswhite.png`
- `walk/patterns/body/pattern12nose.png`
- `walk/patterns/body/pattern1.png`
- `walk/patterns/body/pattern2.png`
- ...
- `walk/patterns/body/pattern23.png`

Это удобно, потому что потом в коде можно будет переключать не набор id, а корень папки для body-слоёв.

## Что можно не дублировать

Если элемент одинаково подходит и к обычной стойке, и к walking-модели, его можно оставить только в старой папке.

Судя по твоему описанию, это относится к:

- `ears/`
- `tails/`
- `manes/`
- `cheeks/`
- tail patterns из `patterns/tail/`

То есть отдельную walking-копию для них сейчас делать не нужно.

## Практический способ работы

Самый простой workflow такой:

1. Скопировать в `walk/` все body-слои из обычной модели.
2. Заменить только те PNG, которые плохо ложатся на walking-позу.
3. Не трогать уши, хвосты и прочие общие элементы.

## Важно

Такая же структура должна поддерживаться и в:

- `site/assets/characters/walk/`

Потому что сайт и игровой клиент сейчас используют разные копии ассетов.
