# Web Export для Fanglaw

В этой папке лежит готовый web-export Godot-клиента.

## Что должно лежать здесь

Минимальный набор файлов:

- `catlaw.html`
- `catlaw.js`
- `catlaw.wasm`
- `catlaw.pck`

Могут быть и дополнительные ассеты Godot export.

## Важное отличие от старых инструкций

Стартовый файл web-версии сейчас называется:

- `catlaw.html`

А не:

- `index.html`

Это важно для:

- локального dev-server сайта
- nginx-конфига
- деплоя на VPS

## Как открыть локально

Локально web-export не нужно открывать напрямую из файловой системы.
Его должен раздавать HTTP-сервер.

Уже готовый вариант:

```bash
cd site
node dev-server.mjs
```

После этого игра доступна по адресу:

- `http://127.0.0.1:4173/catlaw.html`

## Куда выкладывается на сервере

На VPS web-export раздаётся из:

```text
/var/www/fanglaw-web/current
```

## Что важно для браузерного режима

- игра должна открываться по HTTP/HTTPS, а не через `file://`
- для полноценного production-запуска Godot Web нужен `HTTPS`
- если игра и API живут на одном домене, браузерный клиент проще подключается к backend

## Текущее назначение

Эта папка — артефакт сборки для браузерной версии игры.
Она не заменяет Godot-проект в `client/`, а только хранит готовые экспортированные файлы.
